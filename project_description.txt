Project Report: The Automated Mobile App Production Factory
High-Level Goal
The ultimate goal of app-factory-v2 is to create a fully autonomous, end-to-end system that conceives, designs, develops, and publishes complete mobile applications to the Google Play Store with minimal to no human intervention.

It functions as a continuous production line, turning market data into live, published software.

Step-by-Step Operational Blueprint
When finalized and running in production, the app-factory-v2 system will execute the following automated, sequential process:

Phase 1: 💡 Opportunity Discovery
The factory's lifecycle begins with the discovery-cycle-service. This service acts as the market research department. It continuously scans public data sources—like app store trends, social media discussions, and tech news—using the web-scraper-tool. Its goal is to identify underserved niches, emerging trends, or potentially valuable app concepts.

Phase 2: 🧐 Strategic Vetting
Raw ideas generated by the discovery service are passed to the cso-vetting-service. This "Chief Strategy Officer" agent applies a set of high-level business and strategic filters. It evaluates each concept against predefined criteria, such as market size, competitive landscape, and alignment with our core business strategy, discarding unpromising ideas and approving viable ones for further analysis.

Phase 3: 📝 Product Analysis & Specification
Vetted ideas move to the cpo-analysis-service. This "Chief Product Officer" agent performs a deep dive on the approved concept. It defines a detailed product specification, including the core feature set, user interface flow, monetization model, and target audience. The output of this phase is a comprehensive "app blueprint" ready for development.

Phase 4: 👨‍💻 AI-Powered Development
The detailed app blueprint is handed off to the ai-developer-agent-service. This is the core development engine of the factory. Using its GEMINI_API_KEY, it interprets the specification and generates the complete, production-quality source code for the mobile application. Once the code is written, it uses the github-tool (authenticated with the GITHUB_TOKEN secret) to commit the source code to a new, dedicated GitHub repository.

Phase 5: 🚀 Automated Publishing
With the source code created and stored, the cmo-publishing-agent takes over. This "Chief Marketing Officer" agent manages the final release process. It likely triggers a separate build process to compile the app bundle from the new GitHub repository. It then uses the play-publisher-tool and the GOOGLE_PLAY_API_KEY secret to create a new Google Play Store listing, upload the compiled app, and officially publish it for public download.

The entire process is orchestrated through the mcp-gateway ("Master Control Program"), which acts as the central router, directing traffic and data between the various specialized services and tools to ensure a smooth, end-to-end workflow.
